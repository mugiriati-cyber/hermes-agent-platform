## 二、RAG（检索增强生成）方案

### 选型：ChromaDB（轻量，适合服务器）

| 方案 | 优点 | 缺点 | 选？ |
|-----|------|------|------|
| **ChromaDB** | 轻量、Python原生、无外部依赖 | 单机 | ✅ 适合现在 |
| Qdrant | 功能强大 | 要单独跑服务 | ❌ 太重 |
| Milvus | 分布式 | 太重量级 | ❌ |
| Weaviate | 自带UI | 吃内存 | ❌ |
| Pinecone | 云端不用管 | 要钱 | ❌ 但可后期接 |
| **Open WebUI RAG** | 已经有了 | 不灵活 | 🔄 可对接 |

### RAG 架构

```
用户提问
    │
    ▼
┌──────────────┐
│  查询向量化    │  ← DeepSeek Embeddings API (免费)
│  或 BGE 模型  │
└──────┬───────┘
       │
       ▼
┌──────────────┐     ┌─────────────┐
│ ChromaDB     │────►│ 召回Top-5   │
│ 相似度搜索   │     │ 最相关片段  │
└──────────────┘     └──────┬──────┘
                            │
                            ▼
┌──────────────┐
│ 拼入Prompt   │  →  DeepSeek V4 Pro 回答
└──────────────┘
```

**RAG 知识库来源：**
- 用户上传的文档（PDF/Word/MD）
- 对话历史
- 网页抓取的内容
- 系统日志

## 三、智能体集群协同方案

### 核心框架：PraisonAI AgentTeam ✅

```
            ┌─────────────────────────┐
            │     AgentTeam           │
            │  (协调器 / Orchestrator) │
            └────┬────┬────┬────┬────┘
                 │    │    │    │
       ┌─────────┘    │    │    └─────────┐
       ▼              ▼    ▼              ▼
   ┌──────┐     ┌──────┐ ┌──────┐    ┌──────┐
   │Agent │     │Agent │ │Agent │    │Agent │
   │  A   │◄───►│  B   │ │  C   │◄──►│  D   │
   └──────┘     └──────┘ └──────┘    └──────┘
```

### 三种协作模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **顺序流 (Sequential)** | A→B→C→D 依次处理 | 写文章、数据处理流水线 |
| **轮播聊天 (Round-Robin)** | 智能体轮流发言，互相讨论 | 头脑风暴、辩论、游戏 |
| **主从委派 (Hierarchical)** | Hermes 当老大，分配任务给小弟 | 复杂项目、代码开发 |

### 高级功能：Codex 集成

既然 CLI 那边已经导入了 Codex，可以这样用：

| 智能体 | 模型 | 用途 |
|-------|------|------|
| 👑 **Hermes** | DeepSeek V4 Pro | 总指挥、协调 |
| 🛠️ **Codex** | OpenAI Codex | **写代码、搞开发** |
| 🔍 **Researcher** | DeepSeek V4 Pro | 联网搜索 + 资料整理 |
| 🗣️ **小明 (NPC)** | DeepSeek V4 Flash | 角色扮演、聊天 |
| 💬 **小美 (客服)** | DeepSeek V4 Flash | 知识库问答 |

## 四、MCP（Model Context Protocol）

**MCP 就是智能体的「万能插头」** — 让智能体能调任何外部工具。

```
┌──────────────┐     ┌──────────────────┐
│ 智能体       │────►│  MCP 服务器       │
│ 说："查天气" │     │                  │
└──────────────┘     │  ┌────────────┐  │
                     │  │ 天气API    │  │
                     │  ├────────────┤  │
                     │  │ GitHub API │  │
                     │  ├────────────┤  │
                     │  │ 数据库     │  │
                     │  ├────────────┤  │
                     │  │ 文件系统   │  │
                     │  └────────────┘  │
                     └──────────────────┘
```

**可接入的 MCP 工具（开源/免费）：**
| MCP 服务器 | 功能 | 来源 |
|-----------|------|------|
| `mcp-server-filesystem` | 读写文件 | 官方 |
| `mcp-server-github` | 操作 GitHub | 官方 |
| `mcp-server-sqlite` | 查数据库 | 官方 |
| `mcp-server-web-search` | 联网搜索 | 官方 |
| `mcp-server-puppeteer` | 浏览器操作 | 官方 |
| `mcp-server-email` | 收发邮件 | 社区 |
| `mcp-server-redis` | Redis操作 | 社区 |
| `mcp-server-notion` | Notion管理 | 社区 |
| 自定义 MCP | 你想要的任何API | 自己写 |

**Hermes 已经有 MCP 支持了！**（刚才安装 chainlit 时自动装了 `mcp==1.27.2`）

## 六、当前服务器资源评估

| 资源 | 现状 | 够吗？ |
|-----|------|--------|
| CPU 2核 | PraisonAI + FastAPI + Chroma + Open WebUI | 🟡 紧但够 |
| 内存 3.5G + 2G Swap | 同上 | 🟡 紧 |
| 硬盘 30G 剩余 | 代码+向量库+知识库 | 🟢 够 |
| 模型 | DeepSeek API (不占本地算力) | 🟢 够 |

**要升级的：** 当智能体数量 > 10 或 集群并发 > 3 个任务时，建议升到 4核8G。
